About the Project

Project Overview

Comment Sentiment Analysis Using Word Cloud-Based Lexicon

A lightweight and interpretable sentiment analysis system designed for academic use and small-scale real-world applications.

Purpose

This system analyzes user-generated textual comments and classifies them into positive, negative, or neutral sentiments using a lexicon-driven approach. The solution emphasizes transparency, simplicity, and low computational requirements.

Key Features
  • Lexicon-Based Analysis: Uses predefined positive and negative word dictionaries
  • Text Preprocessing: Comprehensive text cleaning and normalization
  • Three-Class Classification: Positive, Negative, and Neutral categories
  • Word Cloud Visualization: Visual representation of frequent terms
  • Dataset Analysis: Batch processing of multiple comments
  • Single Comment Mode: Instant analysis for individual comments
  • Statistical Reports: Detailed sentiment distribution metrics
  • User-Friendly Interface: Clean and intuitive web interface
System Architecture
Core Modules:
  1. Data Loader Module: Handles loading and validation of comment datasets from CSV files
  2. Text Preprocessing Module: Cleans, normalizes, tokenizes text and removes stop words
  3. Sentiment Analysis Engine: Performs lexicon-based sentiment evaluation and classification
  4. Word Cloud Generator: Creates visual word cloud representations based on sentiment
  5. Flask Web Application: Provides user interface and handles routing
Methodology
Sentiment Analysis Process:
1
Input Collection

Comments are loaded from dataset or entered manually by users

2
Text Preprocessing

Cleaning, normalization, tokenization, and stop word removal

3
Lexicon Matching

Tokens are compared against positive and negative word dictionaries

4
Score Calculation

Sentiment score = (Positive word count) - (Negative word count)

5
Classification

Score > 0: Positive | Score < 0: Negative | Score = 0: Neutral

6
Visualization

Results displayed with charts, tables, and word clouds
